RuFaS Outputs Database

Information Here's how you can interact with the stored outputs database:
  1. Hit 'Refresh' to see the result IDs currently stored.
  2. You can do one of three actions:
    • Export data: select one (or more) result IDs to export
      as a group of files (CSV and JSON) to MASM/output/db_output.
      You can also select which of the columns you want to export in
      the CSV files for each of the different tables/output sets.
    • Delete data: select one result ID to delete from the database.
      Warning: you cannot undo this action.
    • Rename data: select one result ID to rename. Type the new
      name into the text box.
  3. Once you have selected the result(s) and an action, hit the 'Enter'
    button at the bottom of the page. A success or failure message
    should appear in red text below the filter box.
Note that the timestamp column is in the format YYYY-MM-DD hh:mm:ss.





Filter Information You can filter the results that you see in the table below by entering the
body of the SQL WHERE clause. For example, if you wanted to see all results
for which the user column is 'bob', you would type user='bob'. You can
filter by any of the columns shown in the table.
Here are some more examples of filters you can apply:
  • result_id > 186
  • title = 'rufas_test'
  • created_at < '2020-07-21 11:30:00' (any time 'less' than a timestamp
    happened before that time, any time 'greater' happened after)
  • version = 'pre_v1' (currently this is the only version available)
  • result_id <= 192 AND created_at > '2020-07-21 11:30:00'
  • created_at BETWEEN '2020-07-21 11:30:00' AND '2020-07-22 11:30:00'
    (specify a range)
  • result_id IN (184, 186, 194) (specify multiple possible values, equivalent to
    result_id = 184 OR result_id = 186 OR result_id = 194)
  • title LIKE 'demo%' (specify a pattern)
Note that when specifying values from the title, created_at, version, and user
columns the values must be in quotes because the columns are of type string.
